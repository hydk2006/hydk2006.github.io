    • 1. 在测试用例中，如何正确调用 Page Object 类的方法来实现测试步骤？

    • 定义Page Object类：为每个页面定义一个对应的Page Object类，并在该类中封装该页面的元素和操作。例如，对于登录页面，可以定义一个LoginPage类，并在其中封装用户名输入框、密码输入框和登录按钮等元素及其操作方法。
    • 实例化Page Object类：在测试用例中，通过传入WebDriver实例来实例化Page Object类。这样，Page Object类就可以使用WebDriver来与页面进行交互。
    • 调用Page Object类的方法：通过实例化的Page Object对象调用其封装的方法来实现测试步骤。例如，调用LoginPage对象的登录方法来执行登录操作。

    2. 如何处理测试用例中的断言和验证逻辑？

    • 使用assert语句：在Python中，可以使用assert语句来检查某个条件是否为真。如果条件为假，则测试将失败。例如，可以断言登录后的页面标题是否与预期相符。
    • 验证页面元素：检查页面上是否存在某个元素，或者某个元素的状态（如是否可见、是否启用等）是否符合预期。这可以通过Selenium提供的元素定位方法和属性访问方法来实现。
    • 比较页面内容：将页面上的内容与预期的内容进行比较。例如，可以抓取页面上的文本或HTML源代码，并与预期的结果进行比较。

    4. 为什么在 PO 框架中需要记录日志？

    • 调试和排查问题：通过记录日志，可以方便地跟踪测试用例的执行过程，了解每个测试步骤的执行情况和结果。当测试用例失败时，可以通过查看日志来快速定位问题所在。
    • 审计和合规性：在某些行业或组织中，可能需要记录测试用例的执行日志以满足审计和合规性要求。通过记录详细的日志信息，可以确保测试用例的执行过程可追溯、可验证。
    • 分析和改进：通过收集和分析测试用例的执行日志，可以了解测试用例的执行效率、稳定性和可靠性等方面的信息。这些信息有助于对测试框架和测试用例进行优化和改进。

    6. 配置并生成测试报告

    1、pytest-html 安装
    pytest-html 是一个 pytest 插件，用于生成 HTML 格式的测试报告。它可以将 pytest 运行的测试结果以 HTML 文件的形式输出，使测试报告更加直观易读

    pip3 install pytest-html

    2、在当前目录下生成测试报告
